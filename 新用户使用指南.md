## 1. 快速开始

### 1.1 三种入门方式

| 方式 | 适合人群 | 说明 |
|------|----------|------|
| 🎯 加载示例数据 | 首次使用 | 包含 5 个常用质粒，快速熟悉功能 |
| 📥 导入现有数据 | 有数据用户 | 从 JSON/CSV/GenBank 导入 |

- 浏览器：Chrome / Edge / Firefox / Safari（最新版本）
- 操作系统：Windows / macOS / Linux
- 存储：至少 10MB 可用空间

## 2. 首次使用

### 2.1 启动引导流程

```
1. 打开 plasmid_manager.html 或 index.html
   ↓
2. 选择开始方式
   ├── 加载示例数据（推荐新手）
   ├── 导入现有数据库
   └── 从零开始
   ↓
3. 完成设置，进入系统
```

### 2.2 加载示例数据

**包含的示例质粒：**

| 质粒名称 | 类型 | 主要特征 | 用途 |
|----------|------|----------|------|
| pUC19 | 克隆载体 | AmpR, lacZ | 分子克隆 |
| pET-28a | 表达载体 | His, T7 promoter | 蛋白表达 |
| pLKO.1 | RNAi | U6, shRNA | 基因敲低 |
| pCDH | 慢病毒 | EF1α, GFP | 基因过表达 |
| lentiCRISPR-v2 | CRISPR | Cas9, sgRNA | 基因编辑 |

**示例数据用途：**
- 了解系统功能
- 测试搜索筛选
- 练习编辑操作
- 学习字段结构

### 2.3 导入现有数据

**支持的格式：**

#### JSON 格式
```json
{
  "plasmids": [
    {
      "name": "pUC19",
      "长度": 2686,
      "保存位置": "E:/plasmids",
      "项目": "Vector Library",
      "大肠杆菌抗性": ["Amp"],
      ...
    }
  ]
}
```

#### CSV 格式
| name | 长度 | 项目 | 大肠杆菌抗性 |
|------|------|------|------------|
| pUC19 | 2686 | Vector Library | Amp |
| pET-28a | 5369 | Expression | Kan |

#### GenBank (.gb / .gbk)
系统自动提取：
- 序列
- 特征（CDS, promoter, etc.）
- 注释信息

### 2.4 从零开始

**最小必需字段：**
- `name` - 质粒名称
- `长度` - 序列长度（bp）
- `项目` - 所属项目

**可选字段：**
- 保存位置
- 持有人
- 抗性标记
- 启动子
- 特征描述等

## 4. 常见问题

### Q1: 如何导入现有数据？

**JSON 文件：**
1. 点击「📥 导入」
2. 选择 JSON 格式
3. 选择文件
4. 确认导入

**CSV 文件：**
1. 确保 CSV 包含 `name` 和 `长度` 列
2. 导入后检查字段映射

### Q2: 数据保存在哪里？

- **本地存储**：localStorage（浏览器内）
- **文件存储**：plasmid_database.json
- **推荐**：使用文件存储，数据更安全

### Q3: 如何与 SnapGene 集成？

1. 在「设置」中配置 SnapGene 路径
2. 点击质粒卡片上的「🔬」按钮
3. 自动调用 SnapGene 打开

### Q4: 字段可以自定义吗？

是的！系统支持自定义字段：
- 预定义字段（不可删除）：name, 长度, 项目等
- 自定义字段：添加任意字段

### Q5: 多人协作可以吗？

当前版本为单机使用：
- 数据存储在本地
- 如需协作，建议：
  - 定期导出 JSON 备份
  - 手动同步数据文件
