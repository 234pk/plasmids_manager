质粒管理系统使用说明书

Plasmid Manager - 实验室科研人员的高效助手

1. 软件简介

质粒管理系统 是一款专为实验室科研人员设计的桌面端质粒数据库管理工具。它旨在解决实验室质粒文件散乱、查找困难、信息记录不全等痛点，通过智能识别、批量导入和结构化管理，帮助实验室建立规范、高效的质粒资源库。

📦 软件版本说明

本次发布包包含三种不同格式的程序，请根据需求选择：

  • 安装版 (Setup.exe): 推荐使用。标准的 Windows 安装程序，会自动安装到系统并创建桌面快捷方式。
  • 便携版 (单文件.exe): 无需安装，双击即用。适合拷贝到 U 盘随身携带。
  • 解压版 (win-unpacked): 文件夹格式的程序包，直接运行其中的 PlasmidManager.exe 即可。
📂 智能批量导入

支持拖拽文件夹直接导入，系统自动从文件名中识别抗性、启动子、标签、物种等关键信息。

📝 自动属性识别

内置智能算法，无需手动输入即可提取质粒特征（如 AmpR, CMV, EGFP, Flag 等）。

🔍 高效检索

支持按文件名、靶基因、抗性、项目等多维度组合搜索，支持拼音首字母快速查找。

📊 结构化管理

按“项目”、“持有人”、“保存位置”对质粒进行分类统计，清晰掌握实验室资源分布。

🧬 序列关联

自动关联 .dna / .gb / .seq 等序列文件，支持调用外部软件直接打开查看。

2. 快速入门

2.1 启动软件

双击运行 质粒管理系统.exe 即可进入主界面。

  • 数据存储: 软件默认在运行目录下的 data 文件夹中存储数据库 (plasmids.json) 和配置信息。请定期备份该文件夹以防数据丢失。

2.2 界面概览

  • 左侧导航栏: 切换不同视图（质粒库、项目管理、持有人管理、设置）。
  • 顶部工具栏: 包含搜索框、批量导入按钮、新建质粒按钮等常用操作。
  • 主列表区: 显示质粒条目，支持排序和筛选。
  • 右侧详情页: 点击列表项可查看质粒的详细信息、序列文件路径及备注。

3. 核心功能指南

3.1 批量导入 (推荐建库方式)

这是初始化实验室数据库最高效的方式。

  1. 点击顶部 “批量导入” 按钮。
  2. 选择文件: 点击区域选择文件，或直接将包含质粒文件的文件夹拖入虚线框内（支持递归扫描子目录）。
  3. 智能识别: 系统会自动分析文件名，提取特征。
    提示: 文件名越规范（如 pCDH-CMV-MCS-EF1-GFP-Puro.dna），识别越准确。
  4. 审核与校准:
    • 进入审核页面，您可以预览识别结果。
    • 标签系统: 点击属性下方的标签（如 AmpR）可快速勾选/取消。
    • 手动补录: 在输入框中直接输入新属性（如 KanR），无需按回车，点击保存时会自动记录。
    • 荧光蛋白: 专门的荧光蛋白字段（如 EGFP, mCherry）现已支持独立编辑。
  5. 确认导入: 点击底部 “确认导入”，数据将写入数据库。

3.2 单条新建

适用于日常零星添加。

  1. 点击 “新建质粒” 按钮。
  2. 填写基本信息（文件名、载体类型、靶基因等）。
  3. 可以直接拖入序列文件到窗口中进行关联。
  4. 点击 “保存”。

3.3 搜索与筛选

  • 全局搜索: 在顶部搜索框输入关键词（如 GFP 或 张三），实时显示匹配结果。
  • 高级筛选: 点击搜索框右侧的漏斗图标，可按“抗性”、“载体类型”、“物种”等特定字段精确过滤。

3.4 编辑与更新

  • 在列表中点击质粒名称打开详情页，点击右上角 “编辑” 图标即可修改信息。
  • 注意: 所有的修改都会实时更新“描述”字段，确保信息的完整性。

4. 常见问题 (FAQ)

Q: 如何备份数据？
A: 请定期复制软件目录下的 data 文件夹到安全位置（如移动硬盘或云盘）。
Q: 识别结果不准确怎么办？
A: 可以在“设置”中自定义识别规则，添加实验室特有的命名习惯关键词。同时，建议统一实验室的质粒文件命名规范（例如：载体-基因-标签-抗性）。
Q: 为什么输入框输入后没有反应？
A: 最新版本已优化交互，输入内容后无需按回车键。直接点击保存，系统会自动将您输入的内容作为新标签提交。
Q: 找不到“荧光蛋白”选项？
A: 请检查是否为最新版本。最新版已在批量导入审核表中补充了“荧光蛋白”独立列。

5. 版本更新摘要 (2026-02-08)

  • 交互优化: 全面移除了输入框对回车键的依赖，防止误操作，输入体验更流畅。
  • 功能补全: 批量导入审核界面新增“荧光蛋白”列，抗性字段拆分为“大肠杆菌抗性”和“哺乳动物抗性”。
  • 数据安全: 增强了自动保存逻辑，防止因未按回车导致的临时输入丢失。
